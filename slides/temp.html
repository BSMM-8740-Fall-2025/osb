MC Methods

Suppose we need to compute an expectation ùîº[g(Z)]\mathbb{E}[g(Z)] for some random variable ZZ and some function g:‚Ñù‚Üí‚Ñùg:\mathbb{R}\to\mathbb{R}. Monte Carlo methods avoid doing any integration or summation and instead just generate lots of samples of ZZ, say z1,z2,‚Ä¶,zMz_1,z_2,\ldots,z_M and estimate ùîº[g(Z)]\mathbb{E}[g(Z)] as 1M‚àëi=1Mg(zi)\frac{1}{M}\sum_{i=1}^Mg(z_i). The law of large numbers states that this sample mean should be close to ùîº[g(Z)]\mathbb{E}[g(Z)].

Said another way, Monte Carlo replaces the work of computing an integral (i.e., an expectation) with the work of generating lots of random variables.

Stochastic processes

A stochastic process, which we will usually write as (Xn)(X_n), is an indexed sequence of random variables that are (usually) dependent on each other.

Each random variable XnX_n takes a value in a state space ùíÆ\mathcal S which is the set of possible values for the process. As with usual random variables, the state space ùíÆ\mathcal S can be discrete or continuous. A discrete state space denotes a set of distinct possible outcomes, which can be finite or countably infinite. For example, ùíÆ={Heads,Tails}\mathcal S = \{\text{Heads},\text{Tails}\} is the state space for a single coin flip, while in the case of counting insurance claims, the state space would be the nonnegative integers ùíÆ=‚Ñ§+={0,1,2,‚Ä¶}\mathcal S = \mathbb Z_+ = \{0,1,2,\dots\}.

Stochastic processes

Further, the process has an index set that puts the random variables that make up the process in order. The index set is usually interpreted as a time variable, telling us when the process will be measured. The index set for time can also be discrete or continuous. Discrete time denotes a process sampled at distinct points, often denoted by n=0,1,2,‚Ä¶n = 0,1,2,\dots, while continuous time denotes a process monitored constantly over time, often denoted by t‚àà‚Ñù+={x‚àà‚Ñù:x‚â•0}t \in \mathbb R_+ = \{x \in \mathbb R : x \geq 0\}.

Markov property

Think of a simple board game where we roll a dice and move that many squares forward on the board. Suppose we are currently on the square XnX_n. Then what can we say about which square Xn+1X_{n+1} we move to on our next turn?

  • Xn+1X_{n+1} is random, since it depends on the roll of the dice.
  • Xn+1X_{n+1} depends on where we are now XnX_n, since the score of dice will be added onto the number our current square,
  • Given the square XnX_n we are now, Xn+1X_{n+1} doesn‚Äôt depend any further on which sequence of squares X0,X1,‚Ä¶,Xn‚àí1X_0, X_1, \dots, X_{n-1} we used to get here.

The third point is called the Markov property or memoryless property. We say ‚Äúmemoryless‚Äù, because we only need to remember what square we‚Äôve reached, not which squares we used to get here. The stochastic process before this moment has no bearing on the future, given where we are now. A mathematical way to say this is that ‚Äúthe past and the future are conditionally independent given the present.‚Äù

Consider the following simple random walk on the integers ‚Ñ§\mathbb Z: We start at 00, then at each time step, we go up by one with probability pp and down by one with probability q=1‚àípq = 1-p. When p=q=12p = q = \frac12, we‚Äôre equally as likely to go up as down, and we call this the simple symmetric random walk.

The simple random walk is a simple but very useful model for lots of processes, like stock prices, sizes of populations, or positions of gas particles. (In many modern models, however, these have been replaced by more complicated continuous time and space models.) The simple random walk is sometimes called the ‚Äúdrunkard‚Äôs walk‚Äù, suggesting it could model a drunk person trying to stagger home.

So far we‚Äôve seen a a few examples of stochastic processes in discrete time and discrete space with the Markov memoryless property. Now we will develop the theory more generally.

To define a so-called ‚ÄúMarkov chain‚Äù, we first need to say where we start from, and second what the probabilities of transitions from one state to another are.

In our examples of the simple random walk and gambler‚Äôs ruin, we specified the start point X0=iX_0 = i exactly, but we could pick the start point at random according to some distribution Œªi=‚Ñô(X0=i)\lambda_i = \mathbb P(X_0 = i).
